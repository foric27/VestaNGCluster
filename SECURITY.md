# Security Policy

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| latest main | ✅ Активная поддержка |
| предыдущие | ❌ Только критические обновления |

## Как сообщить об уязвимости

**НЕ создавайте public issue для security problems.**

Вместо этого:
1. Напишите на email разработчика (указан в профиле GitHub)
2. Опишите проблему подробно
3. Дайте время на исправление перед публикацией

## Политика по secrets

- Signing secrets (keystore, пароли) **не хранятся** в репозитории
- Используйте GitHub Secrets для CI/CD
- Локальная подпись через `keystore.properties` (добавлен в `.gitignore`)
- Бинарные keystore файлы (`.jks`, `.keystore`) добавлены в `.gitignore`

## История security incidents

### 2026-05-30: Утечка signing key

- **Что произошло**: signing keystore и пароли попали в git history
- **Действия**: 
  - Удалены из истории через `git filter-repo`
  - Старый ключ считается скомпрометированным
  - Требуется генерация нового keystore и обновление GitHub Secrets

## Рекомендации

1. Генерируйте новый keystore для каждого major release при необходимости
2. Храните пароли в password manager, не в коде
3. Используйте `--force-with-lease` при force-push после истории
