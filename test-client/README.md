# Тестовый UDP-приёмник

Python-утилита для приёма и визуального отображения UDP-стрима от Android-приложения **VestaNGClusterFlowStudio**.

## Что принимает

| Поток | Порт | Формат |
|-------|------|--------|
| Видео | `5004` | Raw H.264 Annex B (NAL units, start code `00 00 00 01`) |
| Статус | `5001` | JSON UTF-8 (`{"vid":"...","time":"...","lang":"..."}`) |

По умолчанию приложение шлёт на `192.168.40.2`, но скрипт слушает на
`0.0.0.0` — достаточно, чтобы компьютер был в той же подсети.

## Зависимости (Windows)

```powershell
# Если в консоли отображаются иероглифы вместо русского текста,
# переключите кодировку на UTF-8:
chcp 65001

# Установка из корня проекта
python -m pip install -r test-client/requirements.txt

# Или напрямую
python -m pip install opencv-python Pillow
```

## Запуск

```powershell
# Базовый запуск — открывает окно с трансляцией
python test-client/udp_player.py

# Режим только логов — без окна, статистика в консоль
python test-client/udp_player.py --headless

# Указать конкретный IP-интерфейс
python test-client/udp_player.py --host 192.168.40.2

# Изменить порты (если переопределены в RuntimeConfig)
python test-client/udp_player.py --video-port 5004 --status-port 5001
```

## Управление

- **q** или **Esc** — закрыть окно.

## Что показывает

- **Видеопоток** в окне OpenCV (или в консоли в режиме `--headless`).
- **Overlay** с информацией:
  - Разрешение кадра.
  - Текущий FPS (кадров в секунду).
  - Битрейт (оценочный).
  - Режим кластера (`vid`).
  - Время синхронизации (`time`).
  - Язык (`lang`).
  - Цвет полей `time`/`lang` меняется по свежести: белый мигание при обновлении, зелёный (<1с), жёлтый (1–3с), красный (>3с).
- **Ошибки потока** красным цветом:
  - `ПОТЕРЯ ПОТОКА` — если кадры не приходят более 1 секунды.
  - `НИЗКИЙ FPS` — если FPS упал ниже 10.
  - `ПРОПУЩЕНО КАДРОВ: N` — количество подряд пропущенных кадров.

### Режим `--headless`

Выводит статистику в консоль раз в секунду в формате:
```
[ЧЧ:ММ:СС.мс] СТАТ: 1920x640 | FPS=30.0 | Битрейт=8.50 Мбит/с | Режим=nav | Время=143022+0300 | Язык=ru | Возраст_статуса=0.2с
```
Выход только по `Ctrl+C`.

## Поведение

- При запуске **сразу открывает окно в нативном разрешении видеопотока** и ожидает появления UDP-потока.
- Пока видеопакетов нет, скрипт не открывает `cv2.VideoCapture`, чтобы FFmpeg не блокировал главное окно.
- Перед подключением используется быстрый UDP-пробник: OpenCV запускается только после первого пакета на видеопорту.
- Если поток прерывается — **не закрывается**, а продолжает ожидать восстановления.
- Low-latency режим: FFmpeg/OpenCV запускается с `fflags=nobuffer`, маленьким UDP FIFO, `probesize=131072` и `analyzeduration=2000000` для корректного поиска SPS/PPS.
- Первые 10 кадров после подключения пропускаются для стабилизации H.264 декодера.
- Если кадры приходят быстрее, чем окно успевает их показать, скрипт пропускает накопленные старые кадры и выводит самый свежий.
- Overlay рисуется на области 520×N пикселей с шрифтом 16, чтобы вместить все строки включая «Язык».
- Для защиты от зависаний decoder: если `cap.read()` занимает больше 1.5 секунд, захват принудительно пересоздаётся.
- `VideoCapture` открывается совместимым вызовом `cv2.VideoCapture(url, cv2.CAP_FFMPEG)`: некоторые сборки `opencv-python` не поддерживают список параметров в `.open()` и падают с `unsupported parameters`.

## Совместимость

Проверено на Windows 10/11 с Python 3.10+, `opencv-python` и `Pillow` с PyPI.
FFmpeg-бэкенд в составе `opencv-python` обрабатывает UDP H.264 потоки.
Для минимальной задержки параметры FFmpeg задаются внутри скрипта через
`OPENCV_FFMPEG_CAPTURE_OPTIONS` до импорта `cv2`.
Дополнительные параметры конструктора `VideoCapture` не используются из-за
несовместимости с частью Windows-сборок `opencv-python`.
Русский overlay рисуется через Pillow только в области информационной панели,
без полной перекодировки каждого кадра.
